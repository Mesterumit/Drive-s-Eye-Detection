transform = transforms.Compose([
    transforms.Grayscale(),           # convert to grayscale
    transforms.Resize((64, 64)),      # resize all images
    transforms.ToTensor(),            # convert data to PyTorch tensor
    transforms.Normalize([0.5], [0.5]) # normalize
])

 transforms.Grayscale() --> Colored image (3 channels: R,G,B)  →  Gray image (1 channel)
 Eyes open/closed doesn't need color info → simpler and faster

transforms.ToTensor() --> Image (pixels 0-255)  →  PyTorch Tensor (values 0.0 - 1.0)
 Model only understands numbers/tensors, not images

 transforms.Normalize([0.5], [0.5]) --> Values (0.0 - 1.0)  →  Values (-1.0 to 1.0)
Helps the model learn faster & more stable

transforms.Compose() just chains all these steps together so they run automatically one after another on every image.

With zero_grad():

Batch 1:
zero_grad()  → gradients = [0, 0, 0]  (clean!)
backward()   → gradients = [0.5, 0.3, 0.8]  (only batch 1)
step()       → update model correctly ✅

Batch 2:
zero_grad()  → gradients = [0, 0, 0]  (clean again!)
backward()   → gradients = [0.2, 0.1, 0.4]  (only batch 2)
step()       → update model correctly ✅


Order always must be:

1. zero_grad()   → clean whiteboard
2. forward()     → get predictions
3. backward()    → calculate gradients
4. step()        → update weights
